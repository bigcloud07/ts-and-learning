### 1. 현재 TypeScript 경험

- TS를 써본 적이 있는가? 있다면 어느 정도?
  > 이전에 진행한 React 프로젝트에서 사용했었다. 하지만 AI의 도움을 많이 받았고 명확히 알고쓴다기 보다 구현에 딱 필요한 만큼 공부를 하고 써서 누군가 나에게 TS에 대해서 깊게 질문한다면 제대로 답변이 어려울 거 같은 정도의 수준이다. 이번 미션을 진행하면서 해당 부분에 대한 갈증을 어느정도 해소하고 싶다.

### 2. 기존에 새로운 개념을 학습할 때 어떻게 했나요? (수업시간에 v1 떠올렸던 것 첨부)
  ```
  이번에 원정대 활동을 하며 진행한 스코프, 실행컨텍스트, 클로저의 개념 학습법입니다.

  1. 모던자바스크립트 책에서 공부하고자 하는 Chapter의 내용을 쭉 읽어본다.
  2. 읽으면서 모르는 부분이 있으면 머릿속에 생각 or notion에 메모만 하고 탐구하지 않고 쭉 읽어본다.
  3. 다시 한 번 읽으면서 모르는 부분을 AI에게 물어보거나 크루에게 물어본다.
  4. 마지막으로 읽으면서 머릿속으로 구조화한다.
  5. AI에게 관련 개념 문제를 요구하고 풀어본다.
  6. 틀렸다면 왜 틀렸는지 생각해보고 AI 생각과 비교해본다.
  ```


## 학습 로그 #1

**시간**: 03/25 10:50 ~ 14:14 (약 144분, 점심 시간 1시간 제외)
**학습 범위**: typescript-exercises #1~#3

### 1. 첫 학습 전략 설계
  ```
  1. 문제 읽을 때 `Exercise` 부분에 나와있는 문제 요구사항에 집중해서 읽는다.
  2. 어떤 부분을 고쳐야 하는지 예상해본다.
  3. 하단에 있는 문제 해결에 필요한 개념을 담고 있는 Typescript 공식 문서를 찾아본다.
  4. 해당 내용을 읽어보고 모르는 부분이 있으면 AI에게 물어본다.
  5. 문제를 풀어본다.
  6. 가지고 있는 지식을 이용해 해결한 부분(추론을 해서 문제를 해결한 부분)이 있다면 AI에게 정확히 공식 문서의 어떤 부분을 참조해야 하는지 물어본다.
  7. 해당 부분을 읽어보고 몰랐던 개념을 Notion에 나의 언어로 기록한다.
  8. 다음 문제로 넘어간다.
  ```


### 2. 전략 평가

- 효과적이었던 것과 그 이유
  - 문제 요구사항에 대해 집중해서 읽고 어떤 부분이 필요할지 예상을 한 부분(1, 2번) 
  - 아무 생각 없이 정보를 받아들일 때 보다 집중도나 흥미면에서 이점이 많았다.
- 비효과적이었던 것과 그 이유
  - 아직까지 비효율적인 점은 없었다.
### 3. AI 피드백

- 2번 부분, 어디를 고쳐야 할지 예상할 때, 더 구체적으로 생각해라
  >'role' does not exist in type 'User'
  해당 에러 메세지를 보고 `타입 호환성 문제`라고 구체적으로 정의 -> Union Type이 필요하겠다 라는 사고 까지 이어져야 한다고 말한다.
  

### 4. 다음 타임에 바꿀 것

- 유지할 것과 그 이유
  - 전부 유지한다. 현재 나와있는 문제를 해결하기가 어렵지 않았기 때문이다.
- 바꿀 것과 그 이유
  - 전체 학습 과정을 유지하되, 2번의 과정을 진행할 때 AI의 조언대로 문제를 구체적으로 정의하는 과정을 추가하겠다.
  - 해당 과정을 진행하며 `왜?` 라는 질문을 하면서 질문을 하며 읽는 방식이 학습에 더 효과적일거 같다는 생각이 들기 때문입니다.
